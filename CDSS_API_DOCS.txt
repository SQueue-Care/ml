PANDUAN INTEGRASI API CDSS (Clinical Decision Support System)
=============================================================
Dokumen ini disusun sebagai panduan ringkas bagi tim Frontend dan Backend untuk mengintegrasikan fitur rekomendasi penyakit berbasis AI.


BAGIAN 1: PANDUAN UNTUK TIM BACKEND / DEVOPS
--------------------------------------------
Tugas utama tim backend adalah memastikan aplikasi berjalan dan terhubung dengan server Google Gemini tanpa kendala.

1. Kebutuhan Environment Variable
Agar fitur analisis AI bekerja, server wajib memiliki variabel lingkungan berikut di .env (jika lokal) atau di Application Settings (jika di Azure/Cloud):

GEMINI_API_KEY=AIzaSy_isi_dengan_api_key_asli
GEMINI_MODEL=gemini-2.5-flash

Catatan: Sistem sudah memiliki fitur Auto Fallback. Jika model gemini-2.5-flash menyentuh limit (429 Error), sistem akan otomatis berpindah ke model gemini-flash-latest tanpa memberikan pesan error ke Frontend.

2. Cara Menjalankan Server (Lokal)
Pastikan dependensi diinstal, lalu jalankan server FastAPI dengan perintah berikut:
uvicorn app.app:app --reload

API akan berjalan di alamat lokal: http://127.0.0.1:8000
API Production (Live): https://patient-api-fastapi-g8hsazavdgc7hgeu.southeastasia-01.azurewebsites.net


BAGIAN 2: PANDUAN UNTUK TIM FRONTEND
------------------------------------
Tugas tim frontend adalah mengirimkan data keluhan pasien dan menampilkan hasil JSON dari AI ke dalam bentuk antarmuka (UI) interaktif.

1. Endpoint Utama
URL: https://patient-api-fastapi-g8hsazavdgc7hgeu.southeastasia-01.azurewebsites.net/cdss/recommend
Method: POST
Headers: Content-Type: application/json

2. Request Body (Payload)
Tim frontend harus mengirimkan JSON dengan minimal parameter gejala. Umur dan jenis kelamin bersifat opsional namun sangat direkomendasikan untuk akurasi medis.

Format Request (JSON):
{
  "gejala": "perut saya panas melilit, sering mual setelah makan",
  "umur": 30,
  "jenis_kelamin": "L"
}

3. Response Data (Berhasil 200 OK)
Berikut adalah struktur balasan dari server yang sudah disesuaikan dengan desain UI frontend.

Format Response (JSON):
{
  "gejala_teridentifikasi": [
    "perut panas melilit",
    "mual setelah makan"
  ],
  "kandidat_diagnosis": [
    {
      "nama_penyakit": "Gastritis",
      "tingkat_urgensi": "LOW",
      "confidence": 85,
      "departemen": "PENYAKIT DALAM",
      "penjelasan": "Gejala perut panas dan mual setelah makan sangat khas untuk radang lambung (gastritis).",
      "pemeriksaan_lanjutan": [
        "Pemeriksaan Fisik Abdomen",
        "Endoskopi (jika gejala persisten)"
      ]
    }
  ],
  "catatan_medis": "Disarankan pasien menghindari makanan pedas dan asam. Rujuk ke poli penyakit dalam jika nyeri memberat.",
  "disclaimer": "Hasil ini merupakan rekomendasi berbasis AI dan BUKAN diagnosis medis. Keputusan klinis tetap sepenuhnya berada di tangan dokter yang menangani.",
  "status": "success"
}

4. Penjelasan Mapping UI untuk Frontend
Tim Frontend dapat memetakan parameter JSON di atas ke komponen UI dengan aturan berikut:

- gejala_teridentifikasi (Array teks): 
  Gunakan untuk looping sebagai komponen chip/badge kecil berwarna hijau di bawah textarea pasien.

- nama_penyakit (Teks): 
  Gunakan sebagai judul besar (Header) di dalam Card kandidat diagnosis.

- tingkat_urgensi (Teks): 
  Menentukan warna badge di sudut kanan atas Card. Nilainya selalu salah satu dari: "LOW" (Hijau), "MEDIUM" (Kuning), "HIGH" (Merah).

- confidence (Angka Bulat): 
  Angka 1 - 100. Gunakan untuk mengisi persentase lebar (width) pada Progress Bar / Loading Bar.

- departemen (Teks): 
  Gunakan sebagai teks informasi rujukan poli di bagian bawah penjelasan.

- penjelasan (Teks): 
  Gunakan sebagai teks paragraf singkat di dalam card.

- catatan_medis (Teks): 
  Teks ringkas yang bisa diletakkan di footer/bagian peringatan peringatan di bawah daftar diagnosis.

5. Health Check Endpoint
Jika frontend butuh memastikan apakah fitur AI sedang aktif atau mati (misalnya untuk disable tombol AI), silakan gunakan endpoint GET:
URL: https://patient-api-fastapi-g8hsazavdgc7hgeu.southeastasia-01.azurewebsites.net/cdss/health
Method: GET
Response Status: "healthy" (Aktif) atau "not_configured" (API key belum di set oleh backend).
